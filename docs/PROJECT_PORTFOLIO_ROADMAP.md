# FINORA portfolio roadmap

## Phase 1 — Research project

Validate forecasting mathematics, FINORA-KD-Q, grounded RAG, calibrated uncertainty,
drift, graph contagion, cost-aware backtesting, bias and reproducibility. Exit requires
versioned evidence; it does not grant production approval.

## Phase 2 — GitHub project

Maintain CI, issue/PR templates, reproducible environments, release notes, dependency
and image scanning, model cards, documented APIs, and fail-closed external validation.

Suggested project-board views:

1. Data Platform
2. Forecasting Engine
3. RAG and Reasoning
4. Risk and Backtesting
5. UI and API
6. Security and Compliance
7. Deployment and MLOps

Fields: status, owner, milestone, risk level, data classification, validation gate,
reproducibility hash, and approval requirement.

## Phase 3 — Project portfolio

- **Project A — FINORA Engine:** governed data/model platform and enterprise APIs.
- **Project B — Corporate:** legal entity, licensing, privacy, compliance, funding,
  insurance, vendor risk and model governance.
- **Project C — Go-to-market:** validated ICP, pilots, pricing, marketing, sales,
  support and B2B acquisition without overstated performance claims.

## Milestones

- M1 Research baseline: real provider evidence and approved calibration datasets.
- M2 Model validation: governed weights, regime/OOS/bias/security benchmark package.
- M3 Staging: multi-replica load, failure, DR, observability and rollback drills.
- M4 Governance: signed model-risk, security, data-owner and compliance approvals.
- M5 Portfolio pilot: bounded, monitored decision-support pilot; no autonomous trading.
